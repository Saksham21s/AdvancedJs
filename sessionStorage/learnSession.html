Session Storage is a web storage API that allows you to store key-value pairs temporarily in a web browser. Data stored in session storage is available only for the duration of the page session and is cleared when the tab is closed.

ðŸ›  Key Facts About Session Storage:

  • Stores data in key-value pairs.
  • Data is lost when the tab is closed.
  • Stored data is available only on the same origin (protocol + domain + port).
  • Storage limit is around 5MB per domain.
  • Data can be accessed using `sessionStorage.getItem("key")`.
  • To remove a specific key: `sessionStorage.removeItem("key")`.
  • To clear all stored data: `sessionStorage.clear()`.

  • Session Storage is not permanent - Data is lost when the tab is closed.
  • Session Storage is not shared across tabs - Each tab has its own storage.
  • Session Storage is not suitable for large data - For complex data, use databases.
